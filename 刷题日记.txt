由于用例编写过于麻烦没有继续debug了
看官方题解是用了一个Hierholzer算法，算法很简短，我原先逻辑是正着遍历，Hierholzer是利用dfs，最后输出list的逆序
Hierholzer主要代码：
public void dfs(String curr) {
    while (map.containsKey(curr) && map.get(curr).size() > 0) {
        String tmp = map.get(curr).poll();
        dfs(tmp);
    }
    itinerary.add(curr);
}
while (map.containsKey(curr) && map.get(curr).size() > 0)这一句是防止进入死胡同，如果目标地点无法再继续前进，就跳过，然后直接加入list中，这个地点按题意一定是
最后那个访问的地点，然后再while中继续dfs
自己写的比较好的地方：想到了用map，key为String value为一个排序的list 这跟题解思路差不多，还有 map.containsKey(curr) && map.get(curr).size() > 0这一句在自己
写的逻辑中也有体现，并且通过了题目给的简单用例
自己的不足：最后没有debug完没有写完全部，没有想到dfs
心得：学会了求欧拉图和Hierholzer算法，这个算法要记下

这里可能会有人困惑为什么压缩到一维时，要采用逆序。因为在一维情况下，是根据 dp[j] || dp[j - nums[i]]来推d[j]的值，如不逆序，
就无法保证在外循环 i 值保持不变 j 值递增的情况下，dp[j - num[i]]的值不会被当前所放入的nums[i]所修改，当j值未到达临界条件前，
会一直被nums[i]影响，也即是可能重复的放入了多次nums[i]，为了避免前面对后面产生影响，故用逆序。 举个例子，数组为[2,2,3,5]，
要找和为6的组合，i = 0时，dp[2]为真，当i自增到1，j = 4时，nums[i] = 2,dp[4] = dp[4] || dp[4 - 2]为true，当i不变，j = 6时,
dp[6] = dp [6] || dp [6 - 2],而dp[4]为true，所以dp[6] = true,显然是错误的。 故必须得纠正在正序情况下，i值不变时多次放入
nums[i]的情况。

endmap:

value:  1

此时countmap：
key:    1 2 3 4 5
value： 0 0 0 1 1
endmap:
key:    3  5
value:  1  1

第四次 遇到4 3有可以加入 加入

value： 0 0 0 0 1
endmap:

503. 下一个更大元素 II
中等题
单调栈 没想到 现在要知道单调栈怎么用了
对于「找最近一个比当前值大/小」的问题，都可以使用单调栈来解决。
暴力的话需要重复遍历两次
但是单调栈的话 如果遇到递减的 可以把他们都保存起来 加入栈顶 然后现在遇到了一个比栈顶大的元素 就弹出 然后不断重复与栈顶比较 如果都比他们大
则栈顶对应的下标的元素都是可以更新的
如 5  4  3  2 6
先保存5 4 3 2 然后遇到了6  6比2 大 更新下标4  6比3大 更新下标3  一直更新回去
https://www.cnblogs.com/cartooon/p/11290784.html
还有一个是栈的使用
栈不用stack 已经过时了 一般用deque 即ArrayDeque或者LinkedList
这个其实都是双端队列 他们都有addfitst addlast pollfirst polllast peekfirst peeklast
如果当成栈来用 则就是往队头操作   如果当成队列来用 就是往队头和队尾操作
5681. 判断一个数字是否可以表示成三的幂的和
中等题 周赛题
一开始傻傻的用回溯遍历所有可能解
正确题解是如果一个数字可以表示成三的幂的和，那么这个数字转换为三进制后，数字应该只有0和1
class Solution {
    public boolean checkPowersOfThree(int n) {
        while (n > 0) {
            if (n % 3 == 2) {
                return false;
            }
            n /= 3;
        }
        return true;
    }
}

   7 111  3&7=3 011
